# Tagging Strategy

Usa una combinazione minima ma coerente di tag. Evita tassonomie troppo profonde.

## Pattern Consigliato

Combina, quando utile:

* `dominio` -> es. `database`, `cfml`, `report`, `fatturazione`
* `capability` -> es. `procedura`, `analisi-tecnica`, `troubleshooting`
* `modulo-o-provider` -> es. `seas`, `mantis`, `office`

## Esempi

* guida SQL di un provider: `database`, `procedura`, `seas`
* analisi di regressione su UI CF: `cfml`, `analisi-tecnica`, `frontend`
* report mensile esportato: `report`, `finanza`, `mensile`

## Regole Pratiche

* Crea prima i tag nel dizionario.
* Non usare tag sinonimi per lo stesso concetto.
* Se un tag non migliora la ricerca o il filtraggio, non aggiungerlo.
