# Deliverable Templates

## 1. Requisiti / Contesto

```md
# [Titolo]

## Contesto
## Requisiti espliciti
## Requisiti impliciti / inferiti
## Vincoli
## Domande aperte
## Evidenze
```

## 2. Specifiche / Fonti

```md
# [Titolo]

## Fonti analizzate
## Specifiche rilevanti
## Cosa emerge dalle fonti
## Pattern riusabili
## Gap o adattamenti attesi
## Evidenze
```

## 3. Stato Dell'Arte

```md
# [Titolo]

## Commit / fonti analizzate
## Componenti toccati
## Cosa e gia implementato
## Configurazione gia gestita
## Punti incompleti o dubbi
## Evidenze
```

## 4. Analisi Tecnica Finale

```md
# [Titolo]

## Sintesi esecutiva
## Fatti accertati
## Inferenze e interpretazione
## Rischi
## Strategia consigliata
## Passi operativi
## Assunzioni
```

## Regole Di Scrittura

- Usa italiano se il contesto e italiano.
- Redigi i segreti.
- Scrivi i file in modo riutilizzabile come handoff tecnico.
- Se una conclusione e inferita, dichiaralo esplicitamente.
